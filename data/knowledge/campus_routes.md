# 校园参观路线分类

本文件用于帮助新生助手区分不同校园参观目的。用户说“参观路线”时，不应默认所有人都是游客，也不应只返回固定路线；应先判断参观目的，再选择对应路线。

## 新生校园熟悉路线

适合：
新生第一次入校，想快速熟悉之后学习和生活中最常用的地点。

路线：
东大门 → 行政楼 → 教学区域 → 包玉刚图书馆 → 学生服务中心 → 食堂 → 宿舍区域

重点：
熟悉教学楼、图书馆、食堂、宿舍、学生事务办理地点和学生活动区域。回答时应偏实用，帮助新生减少开学第一周迷路和办事不熟的问题。

可识别表达：
第一次来交大、熟悉校园、了解学校、入校后怎么逛、开学前熟悉一下、新生想看看校园。

## 校园景点打卡路线

适合：
游客、校友、朋友来访，或用户明确想拍照、打卡、看风景。

路线：
东大门 → 包玉刚图书馆 → 思源湖 → 校园特色建筑 → 樱花大道

重点：
包玉刚图书馆、思源湖、校史相关建筑、校园特色景观、适合拍照的位置。回答时应偏游览体验和拍照打卡，不要把路线写成报到流程。

可识别表达：
适合拍照、打卡、游客参观、校园景点、风景好、带朋友逛、有什么好看的地方。

## 家长陪同参观路线

适合：
家长第一次送孩子到校，想了解孩子未来学习生活环境。

路线：
宿舍区 → 食堂 → 教学楼 → 图书馆 → 学院楼

重点：
学院位置、宿舍生活区、食堂、校园环境、校医院、常用校门和接送点。回答时应关注材料确认、住宿生活、安全和防诈骗，但不要替家长做过度承诺。

可识别表达：
家长、新生家长、送孩子报到、陪同报到、看看孩子未来环境、提前看看学校、接送。

## 参观目的标签

- `freshman_orientation`：新生校园熟悉路线。
- `scenic_tour`：校园景点打卡路线。
- `parent_visit`：家长陪同参观路线。
- `unknown`：目的不明确。若 profile.role 是 parent，优先 parent_visit；否则优先 freshman_orientation，并可以提示用户也可选择拍照打卡路线。
